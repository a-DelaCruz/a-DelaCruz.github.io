    Cross-Build Prerequisites

    For building our own 64-bit kernel fom scratch using the mainstream or the Raspberry kernel, we must have a lnux machine (or Windows 10's Bash shell) with the required packages in order to compile the kernel.

    Built on Ubuntu Server 16.04.03 LTS

        bc      device-tree-compiler            ncurses-dev      
    gcc     gcc-arch64-linux-gnu            libncurses5-dev
    git     pkg-config-aarch64-linux-gnu    qemu-user-static
    make    u-boot-tools

    Compile the Kernel

    Use a Linux machine of your choice or Windows 10‚Äôs Bash Shell to compile the kernel is up to you. To get the kernel we need, we can either download it in a compressed file form using the links above or we could use git to make it simpler.

        [~]$ git clone --depth 1 --branch rpi-4.11.y --single-branch https://github.com/raspberrypi/linux.git folder-name

    Once the download or rather cloning complete, go to folder-name.

    bcmrpi3_defconfig - For the RPi3 kernel config. Then configure it further if you want.
    O=~/output-folder - To place the built files in a different directory.
    -jX - To build the kernel in a multithreaded way. X = [Num of core x 2]

        [~folder-name]$ sudo make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- bcmrpi3_defconfig
    [~folder-name]$ sudo make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- -j4 O=../rpi3-build

    Locate Image file at [~rpi3-build/arch/arm64/boot/] and bcm2710-rpi-3-b.dtb at [~rpi3-build/arch/arm64/boot/dts/]

    • [x] Compile the Kernel üëå


    Create config.txt and cmdline.txt

    For the config.txt and cmdline.txt, we have to create them and include this lines.

    config.txt

        arm_control=0x200
    enable_uart=1
    device_tree_address=0x100
    device_tree_end=0x8000
    dtparam=i2c_arm=on
    dtparam=spi=on

    kernel8.img

    cmdline.txt

        net.ifnames=0 dwc_otg.lpm_enable=0 console=ttyAMA0,115200 console=tty1 root=/dev/mmcblk0p2 rootfstype=ext4 elevator=deadline fsck.repair=yes rootwait
    • [x] Create config.txt and cmdline.txt üëå


    Transfer the necessary files to the MicroSD

    Since we dont have a Ramdisk yet, we are using this initrd.img for now.

    Rename the Image file we had earlier to kernel8.img. start.elf file knows kernel8.img file is a kernel to load.

    MicroSD card should now contain these files below:

        bootcode.bin
    start.elf
    config.txt
    cmdline.txt
    kernel8.img
    initrd.img
    bcm2710-rpi-3-b.dtb
    • [x] Transfer the necessary files to the MicroSD üëå


    Boot to test

    Place the MicroSD on to the Raspberry Pi 3 and power it on. You can now then verify at this point that our kernel boot as well and will stop at some point since we still do not have a filesystem.

    If it did not boot or no display at all, we will use U-Boot.

    • [x] Boot to test üëå
